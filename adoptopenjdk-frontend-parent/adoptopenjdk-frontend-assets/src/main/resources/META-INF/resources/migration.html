Migrate to api.adoptium.net

AdoptOpenJDK has moved to the Eclipse Foundation.

As part of the move to Eclipse the AdoptOpenJDK will no longer produce new builds. It is therefore recommended that clients that require OpenJDK binaries should migrate to using api.adoptium.net as soon as possible.

Clients that require OpenJ9/Semeru builds for now can continue to use AdoptOpenJDK however should look out for the upcoming Semeru api that will be the future delivery channel for Semeru builds.

Terms

  • AdoptOpenJDK - The legacy project
  • api.adoptopenjdk.net - Legacy API of the AdoptOpenJDK project
  • Adoptium - New Eclipse Project
  • api.adoptium.net - New API for the Eclipse Adoptium project
  • Temurin - The (TCK'd and AQA tested) OpenJDK binary produced by the Eclipse Adoptium project. Temurin binaries are the successor to AdoptOpenJDK binaries
  • Semeru Open Edition - The name of the new distribution of the OpenJ9 project (not to confuse with Semeru Certified Edition, which is not related to AdoptOpenJDK)

API compatibility

We have kept the API usage identical between the AdoptOpenJDK V3 API and api.adoptium.net for ease of migration. And we plan to continue to maintain it as backwards compatible with the AdoptOpenJDK API.

Update Vendor List

tl;dr :

  • AdoptOpenJDK API will deliver Eclipse Temurin binaries
  • Eclipse Adoptium API will NOT deliver AdoptOpenJDK binaries

Changes at AdoptOpenJDK

The Vendor list in the AdoptOpenJDK API (at time of writing) is as follows:

adoptopenjdk, openjdk, alibaba, ibm, eclipse

In the past we distributed both Hotspot and OpenJ9 under the adoptopenjdk Vendor and when requests were made that did not explicitly state which Vendor is required, the request would default to adoptopenjdk.

Now that AdoptOpenJDK will no-longer produce binaries, in order to keep delivering updates to those depending on the AdoptOpenJDK API, adoptopenjdk has been made synonymous to eclipse in the AdoptOpenJDK API. Requests that explicitly ask for adoptopenjdk, or do not specify a specific Vendor, will return assets that match both adoptopenjdk and the new eclipse builds. The synonym only works one way, requests asking specifically for eclipse will ONLY return builds produced by Eclipse Adoptium and NOT builds that were created at AdoptOpenJDK.

The addition of the ibm vendor is to support the new Semeru project who will be distributing OpenJ9 builds.

Soon with the exception of legacy binaries, most requests for Hotspot JVMs should ultimately return an Eclipse Temurin binary.

Changes at Eclipse Adoptium

eclipse has become the new default and only Vendor at api.adoptium.net. New releases will have the vendor eclipse. api.adoptium.net will only distribute binaries produced by Eclipse.

Update jvm_impl List

The jvm_impl list in the AdoptOpenJDK API is as follows:

hotspot, openj9, dragonwell

At api.adoptium.net (at time of writing), we are only distributing hotspot binaries, this has therefore become the one and only jvm_impl at api.adoptium.net

Reset stats

Download statistics at Adoptium will only show downloads of Adoptium binaries.

The AdoptOpenJDK API will show download statistics of ONLY AdoptOpenJDK binaries.
